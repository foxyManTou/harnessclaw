# 本地代码执行器决策要点

> 基于调研（详见 `local-code-executor-research.md`），需要领导在以下 3 个决策点上明确选择。

---

## 决策点 1：体积上限

**问题**：单平台安装包体积增量可以接受多少？

| 选项 | 体积增量 | 包含内容 | 体验 |
|------|---------|---------|------|
| A. 无硬性上限 | 90-130MB | Python 本体 + 15 个预装包 + Node + pip install | ✅ 开箱即用，覆盖 80%+ 场景 |
| B. < 100MB | 60-100MB | Python 本体 + 5 个核心包 + Node<br/>或：Python + 15 个预装包（不做 Node） | ✅ 主流场景开箱即用，部分场景需 pip |
| C. < 60MB | 30-50MB | 仅 Python 本体，不预装包 | ⚠️ 任何包都要首次 pip install |
| D. < 30MB | 0 | 检测系统环境 + 按需下载 | ❌ 违反"零环境依赖"约束 |

**对标参考**：Claude Desktop（同类 Electron 应用）已打包 Node（~30MB），正在加 Python，预计总体积与选项 A 一致。

**推荐**：**选项 A**（90-130MB），理由：
- 对标业界同类产品
- 开发者工具从 200MB → 300MB 可接受
- 开箱即用体验最好

---

## 决策点 2：Node 支持

**问题**：首版要不要支持 Node 代码执行？

| 选项 | 体积 | 说明 |
|------|------|------|
| E. 要 Node | +30MB | Python + Node 双支持 |
| F. 不要 Node | 0 | 仅 Python（Agent 用 Node 频率远低于 Python） |

**使用频率参考**：调研显示，AI Agent 执行代码时 Python 占 90%+，Node 场景较少（主要是脚本处理、JSON 操作等）。

**推荐**：**选项 E**（要 Node），理由：
- 补全两类运行时（符合需求原文"两类 python、node"）
- +30MB 增量不大
- 避免后续用户反馈"为什么不支持 Node"

**如果体积高度敏感**：可选 F，首版只做 Python。

---

## 决策点 3：Python 预装包数量

**问题**：Python 要预装多少第三方包？

| 选项 | 预装包数量 | 体积 | 覆盖场景 | 缺包时 |
|------|-----------|------|---------|--------|
| G. 常用 15 个 | pandas、numpy、requests、beautifulsoup4、lxml、matplotlib、openpyxl、pillow、pyyaml、python-docx、pypdf 等 | +30-50MB | 数据分析 + 爬虫 + 图像 + 文件处理（80%+） | 白名单内静默 pip，白名单外审批 |
| H. 核心 5 个 | pandas、numpy、requests、matplotlib、openpyxl | +10-20MB | 数据分析（60%） | 更频繁 pip install |
| I. 不预装 | 0 | 0 | 完全靠 pip（首次使用任何包都要装） | 每个包都要审批（白名单内静默） |

**安全机制**：无论哪个选项，缺包时都有 pip 自动安装（白名单 + 审批双重防护），只是触发频率不同。

**推荐**：**选项 G**（15 个），理由：
- 覆盖 80%+ 场景，pip install 只在长尾场景触发
- 白名单内（Top 100 常用包）静默安装，用户无感知
- 体积增量（+30-50MB）相对收益合理

**如果体积敏感**：选 H（5 个核心包）。

---

## 快速决策组合推荐

### 方案 I（完整版）：A + E + G = 90-130MB
- Python 本体 + 15 个预装包 + Node + pip install
- **开箱即用，覆盖 80%+ 场景**
- 对标 Claude Desktop 的目标形态
- **适合：追求极致体验，体积可接受**

### 方案 II（精简版）：B + E + H = 70-100MB
- Python 本体 + 5 个核心包 + Node + pip install
- 主流场景开箱即用，部分场景需 pip
- **适合：体积略敏感，但仍要主流场景无缝**

### 方案 III（仅 Python）：B + F + G = 60-100MB
- Python 本体 + 15 个预装包，不做 Node
- Python 场景开箱即用，Node 不支持
- **适合：体积敏感，且 Node 场景可以不要**

### 方案 IV（极简版）：C + F + I = 30-50MB
- 仅 Python 本体，不预装包，不做 Node
- 任何包都要首次 pip install
- **适合：体积高度敏感，可接受首次配置**
